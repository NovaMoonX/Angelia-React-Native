Beta update notes

Last updated: 2026-05-16
Branch: beta-v-1.0.7
Related PR: TBD

Use this file as the source for the next beta update modal. Keep it short, warm, and focused on what changed for beta testers.

Current branch highlights:
- The Post Activity **Unread Only** filter now shows a live unread-post count, so it is easier to see at a glance how many posts still need your attention.
- Post Activity unread cards now stay put while you browse: we keep reaction activity in the Unread view for that screen session and only mark those posts as seen after you leave the screen.
- Back navigation is now more predictable: leaving Private Notes or Conversation always returns you to Post Details, and leaving Post Details now always lands on Feed with a clean stack.
- OTA feed prompts are now smarter to target: broadcast messages and feedback forms can now be limited by device type (iOS/Android/all) and app version range, so we can send the right in-app message to the right testers.
- Edit mode now stays focused: the Post Create Reset action is hidden while editing, so the top bar keeps a clean Cancel + Save flow.
- Post Detail now gives authors more room between the edit and delete actions, making those controls easier and safer to tap.
- Audio recording got a big upgrade: you can now record multiple clips (up to your attachment limit), each clip caps at 3 minutes, and the final 15 seconds now show a clear on-screen countdown so you can wrap cleanly.
- iPhone audio recording is now much more reliable: the recorder now flips into recording mode before capture starts, so tapping Start Recording actually kicks off the mic instead of quietly failing.
- In Audio Record, the Preview & Details player now shows each clip’s default label (like Recording 2) when no custom title is entered yet, so it’s clearer which recording you’re editing.
- Caption indicators now use a clean icon badge (instead of emoji) and show consistently on audio, video, and image attachments in composer/post cards/post detail.
- Audio clips in post card carousels now sit vertically centered with an opaque media backdrop, so audio-only carousels no longer feel like they have awkward extra bottom space.
- Carousel audio playback now behaves more naturally: if you swipe to another media item, the previous audio clip stops automatically.
- Fixed a carousel audio edge case that could trigger an ErrorBoundary crash while swiping between items during playback.
- Finished clips can now restart from the beginning with one tap on Play (instead of getting stuck at the end).
- Audio clip editing in Post Create now uses a clean single-line title field (same 60-char limit as recording), and audio thumbnails keep the mic + title centered in the preview square.
- In audio clip edit sheets, Remove now clears both title and caption together for existing clips (so titles no longer linger after removing metadata).
- Post edit saves now honor removed clip metadata reliably: when you clear a title or caption and save, those cleared values stay cleared instead of reappearing from older media data.
- Posting now feels more reliable on slower connections: media uploads are queued and continue processing after you return to Feed, with automatic retry behavior and a stronger fallback path.
- Feed + Post Activity now show live upload progress percentages for in-flight posts, and if your post finishes while the app is backgrounded, we send a local "Your post is live" notice.
- Upload progress is now smoother and only moves forward, so percentages no longer jump backward mid-upload.
- If your upload finishes while you're actively in the app, you now get an instant "Your post is live" toast too.
- Post-live toasts are now easier to dismiss: close button taps and swipe-dismiss gestures are more reliable.
- Swipe-to-dismiss on post-live toasts now catches drag gestures more consistently (even quick short swipes), so closing feels less finicky.
- Once a post-live toast starts dismissing, extra touches are now ignored so it closes cleanly without repeated/ghost interaction behavior.
- Post-live toast dismiss animations are now much quicker, so close taps and successful swipes disappear almost instantly instead of feeling delayed.
- Toast dismiss now has a fallback completion path, so if animation callbacks get interrupted the toast still closes instead of getting stuck on screen.
- Close-button taps on post-live toasts now dismiss instantly, and extra rapid taps during teardown are ignored to prevent the "it won't close" feeling.
- Post-live toasts now require a more intentional drag before swipe takeover, so small finger movement no longer steals the close-button tap and first-tap dismiss feels immediate.
- Post-live toasts now clear faster under heavy upload activity, so close/swipe interactions and auto-dismiss no longer feel stuck while media is still finishing in the background.
- Post-live toast gestures now avoid the noisy native animated update warnings, so swipe/close handling stays smooth without console spam.
- Post-live toast animations now stay on one animation path, which avoids the mixed native/JS driver crash and keeps dismissing reliable.
- Post-live toasts now appear immediately when they mount, so they no longer depend on an entrance animation to become visible.
- Audio players got a visual refresh: waveform bars are smoother and stable (no jumpy block effect), progress feels cleaner, and the play button now sits on the right for a tidier layout.
- The audio waveform is now taller too, so playback feels more readable at a glance.
- Carousel audio cards now place the player a bit higher and give it more height, so it is easier to tap play without accidentally hitting the carousel next button.
- Editing a post now preserves existing attachment URLs reliably (we match by storage object path and keep Firestore’s canonical URL), so original media no longer disappears after save.
- Changing a post to a different circle now re-publishes it to the destination circle timeline (including Daily Circle) so members of that circle receive it promptly.
- Media URLs now survive Camera/Gallery/Audio navigation during post edit (we now safely encode/decode media route payloads), so existing image previews no longer break after adding new media.
- Editing a post now keeps existing media previews visible after round-tripping through Camera/Gallery (for example, adding a new video no longer makes the original image preview disappear).
- Editing posts with media now saves cleanly when attachments don’t have thumbnails (no more silent Firestore "Unsupported field value: undefined" failure).
- Post media edits are now reliable after publish: adding/replacing/removing attachments while editing no longer fails from storage permission issues.
- Opening your **first private note notification** is now stable: no more quick bounce back to Feed before the notes screen settles.
- You can now **edit your posts after publishing** (text, circle, priority, captions, media order, and attachments), and Post Detail shows you an author-only "Last edited" timestamp.
- Single taps on conversation messages still close the keyboard, while double taps now keep it open so editing feels natural on iPhone and Android.
- Double tapping your own conversation messages now keeps the keyboard open, so editing feels smoother and doesn’t kick you out of the composer.
- You can now **double tap your own conversation messages to edit them**, with a one-time tip the first time it’s available and a quick warning before switching away from a draft you were already typing.
- Notification Settings now has toggles for **Reaction** and **Message** notifications, so you can control exactly which post activity pings you — all on by default.
- Post Notification Settings no longer shows circles you own — it only shows circles you've joined, which is where per-circle tuning actually makes sense.
- Each circle in Post Notification Settings now has an **Enable All** switch, and each person group has one too — making it easy to turn everything on or off in one tap without losing your Big News setting.
- Android long-press haptics on post cards are now more reliable, and tapping reaction chips in post detail gives a subtle haptic too.
- In post detail, tapping someone else's reaction chip now adds that same emoji as your reaction in one tap.
- Audio attachment flow is now **in-app recording first**: tapping the mic opens a recording screen, and we no longer route people into file/folder audio picking.
- Post composer now shows quick, dismissible hint cards for captions and media reordering, so these features are easier to discover.
- Private circles now show a clean lock icon in the top-right corner of circle cards in account views.
- Big News / Worth Knowing now show as slim banners under headers in post detail and conversation instead of chunky badges.
- Conversation now supports replies to replies (up to 2 levels), shows a warning when a thread is too deep, and aligns reply connector lines with the parent avatar.
- Reply connector lines in conversation now visually chain from parent to child, so threaded replies read like one connected conversation path.
- Reply connectors now keep the parent thread line visible across sibling replies too, so every reply under the same parent stays connected.
- Reply elbows are rounded again and sibling branches now flow in the right direction from parent to child, so thread lines look cleaner and easier to follow.
- Nested replies now keep ancestor lines alive through deeper branches, so later direct replies still connect cleanly back to the original parent.
- Thread connector segments now overlap cleanly between rows, removing the tiny gaps that could appear in long reply chains.
- Closing Camera, Gallery, or Audio Record now takes you back to Post Create with your in-progress content preserved instead of dumping you to Feed.
- Post Create now auto-saves your draft locally (text, media, circle, priority, status, and hint/banner dismissals) until you cancel or your post uploads successfully.
- Post Create now has a confirmed **Reset** action so you can clear the whole draft in one tap when you really want a fresh start.
- Post Create now shows a dismissible heads-up that gallery video uploads are currently limited, and that recording videos in-app is the reliable path for now.
- On Android, Post Create no longer traps content in a short top-only scroll region — the composer now scrolls as one natural vertical surface.
- The Post Create Reset button now only appears after you actually make changes to the draft.
- When a post host tries to leave Post Detail with unread private notes or unread messages, we now show a heads-up modal that calls out the red indicators and lets them review now, exit anyway, or disable this warning across all posts.
- Post Activity no longer clears unread message/private-note indicators just because a post card becomes visible; those clear only after opening the relevant detail views.
- Viewing Post Detail now clears reaction unread only when you leave that screen; unread private notes/messages stay until you open Private Notes or Conversation.
- Tweaked the unread leave-warning modal spacing so “Don't show this again” sits with clearer separation below “Exit Post Anyway.”
- Replies in conversation now appear right after the message they're responding to, so threads are way easier to follow.
- When replying to a message, the reply banner now shows a short preview of the message you're replying to — no more guessing which message you're responding to.
- Replied messages now show a **quoted preview** directly in the bubble so anyone reading the thread can see the context at a glance.
- Conversation headers for Big News and Worth Knowing posts no longer play entry animations — the header just shows up, clean and immediate.
- Big News / Worth Knowing tier banners in conversation now center their label text, so the priority callout feels cleaner and easier to scan.
- Big News / Worth Knowing tier banners in post detail now center their label text too, so both screens match visually.
- The "Unread Only" filter on Post Activity now shows a small dot when there's unread activity, even before you switch to that tab — so you always know something new is waiting.
- Notification setting toggles (Reactions, Conversation messages) now confirm their new state with a brief success toast, so it's clear the change saved.
- Fixed an iOS bug where the keyboard was covering the caption/title input sheet when adding captions to media in post composer.

Previous branch highlights:
- You can now mark a custom circle as **Private** when creating or editing it — only you can invite people to a private circle, and it won't be suggested to others when they leave a post.
- Private circles show a 🔒 badge so it's always clear which circles are invite-only.
- The My Circles tab now shows a one-time notice explaining the difference between public and private circles, with a "Learn more" sheet for the full breakdown.
- For private circles, the invite link is hidden from members — only the host sees it.

Older highlights:
- Posts now support MP3 audio attachments: you can pick audio files from your phone, post them, and play them right in feed/post detail with a clean audio player UI.
- Haptic feedback added: long-pressing a post gives a subtle pulse, and tapping an emoji reaction gives a light tap — both make the app feel more alive.
- GIFs now actually animate in the feed! Previously they'd be stuck on the first frame; now they play properly.
- When you open the app fresh, it now jumps you straight to the top of your feed instead of leaving you mid-scroll from last time.
- Before posting, you can now reorder your media by long-pressing a photo/video in the strip and using the arrows to rearrange.
- You can now add captions to individual photos and videos before posting — a 📝 badge shows on any item with a caption, and captions display in full when you open the media viewer.
- Status updates now give you two ways to expire: pick a duration (like before) or set an exact future date and time.
- Reactions now sort more consistently across post views: most-used emoji first, then ties break by which emoji appeared first on the post.
- Feed now shows a slim upload progress bar at the top whenever your posts are still uploading, including a count when multiple uploads are in flight.
- Post Activity now includes an Uploading filter so you can quickly see in-progress uploads in one place.
- Worth Knowing and Big News labels now show directly on both Post Details and Conversation headers, so post priority is always clear.
- App update checks now use a runtime app-version constant instead of relying only on cached dismissal values, which prevents showing "new update" prompts to people already on the latest build.
- Conversation alerts are smarter now: you get notified by default when new messages come in, replies can ping the specific person being replied to, and post Hosts no longer get double-pinged when a reply targets them.
- Notification Settings now includes a dedicated Reply Notifications toggle, so you can choose whether direct replies to your messages should send a push.
- Post notifications are now per-circle: each circle has its own toggles for Big News, Worth Knowing, Everyday updates, and posts with attachments, grouped by person so it is easy to tune everyone at once.
- New notification defaults are simpler: by default, users only get Big News notifications, and they can opt into extra activity alerts whenever they want.
- Post-activity alerts now clear at the exact moments you review them: tapping a reaction/message/private-note notification clears that alert, opening Notifications clears new message alerts, and opening Private Notes clears private-note alerts.
- The top card in Notifications is now a one-time release notice for new notification controls, and it disappears after you actually open Notification Settings.
- When that release notice is waiting for you, the feed bell now shows a separate blue dot that clears once you visit Notifications, while the notice card itself stays until you open Notification Settings.
- You can now choose exactly which post activity should ping you: reactions, private notes, and conversation messages each have their own toggle in Notification Settings (all on by default).
- Reaction alerts now avoid spam by cooling down repeats from the same person on the same post for 10 minutes, while still delivering conversation message alerts right away.
- You can now react to posts directly from your Feed — long-press a post to open a quick reaction peel right under it, so reacting feels instant and keeps your place.
- Feed reaction peel is now more compact with smaller emoji chips, so it takes up less vertical space while still staying easy to tap.
- New-post feed notice now checks the current top post after fresh posts arrive, and only auto-dismisses when that new first card is actually fully visible.
- New-post feed notice now keeps its count stable while fading out, so you never see a quick “0 new posts” flash.
- New-post feed notice now waits for a real scroll-back-to-top before dismissing after it appears, so it no longer vanishes instantly.
- When you open the feed emoji picker, we now keep the reaction peel visible, nudge the selected post higher, and show a much shorter picker (about half the previous height) so more of the post stays in view.
- Reaction suggestions now prioritize emojis other people already used on that post, while still keeping the familiar common emoji options in the mix.
- Post Activity now keys off your local app-open time, so you only see truly new activity (no more giant “everything is new” moments after updates or sign-ins).
- You can now pull to refresh in both Feed and Post Activity whenever you want a quick manual sync.
- Deleted posts now disappear cleanly from live lists, and if a post vanishes while you are viewing it (or its conversation), you now get a friendly “post may be deleted” message with a one-tap path back to Feed.
- Conversation “joined” notices now appear only when someone actually sends their first message, so the chat timeline feels cleaner.
- Emoji picker now lets you type and submit any emoji directly, so missing picker entries (like 🧶 or 🪡) are no longer blockers.
- Feed now shows posts from all your connected friends — we fixed a sneaky permissions issue that was silently hiding some posts from the feed.
- You can now sort your post activity by newest or oldest posts — giving you more control over how you browse your activity.
- Post Activity now auto-jumps to your unread activity when you open the screen — so you see what matters right away.
- Your uploaded avatar photo now shows on the compose screen — no more missing profile pictures!
- Before posting, you'll now see a quick 5-second countdown with your circle name and attachment count — so you can catch any mistakes. Cancel it or tap "Post Now" to skip the wait.
- We can now send you a message right inside the app (info, warning, or important heads-up) — no app update needed.
- We can now prompt you to fill out a feedback form from inside the app, and we can update the form link anytime without a new release.
- App update prompts are now part of the same priority system as all other feed modals — so they never compete or stack on top of each other unexpectedly.
- Status emoji picker now works correctly on iPhone — tapping the emoji in the status sheet opens the picker as expected.
- Post-activity alert dots now clear more naturally: tapping a notification or opening the relevant post/conversation/notes view will mark those reaction/message alerts as seen.
- Post Activity now marks items as seen only when that post card is fully visible on screen (or when you open the post details screen), so unread highlights no longer disappear just from opening the screen.
- Emoji picker now validates typed input before letting you submit: valid emoji can be used instantly, and non-emoji text shows a clear inline error state.
