Use this folder for plain-text release notes tied to production versions.

Each file should be named exactly after the semantic version in `app.config.js`, for example:

- `versions/1.0.1.txt`
- `versions/1.2.0.txt`

Each file should contain simple bullet lines describing the release updates, changes, and improvements.

Example:

Branch: release/1.0.1
- Bug fixes & reliability improvements for Daily Circle feed sync
- Updated release alert behavior
- Minor onboarding copy polish
